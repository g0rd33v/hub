Part 1 — Hub

What Hub is

Hub is a platform for running digital products — bots, sites, APIs — from a single place. You bring your credentials. Hub does the rest: hosting, versioning, dashboard, analytics, AI-powered build.

Platform
Run anything from one place
Telegram bots, static sites, API services, mini-apps. Every project gets a public URL, a private dashboard, a versioned file system, and analytics out of the box.
Build
AI builds the project for you
Describe what you want. Hub generates a build prompt, drops it at your project URL. Open it in Claude for Chrome — Claude reads it and ships the finished project automatically.
Access
Three-tier access model
SAP (server owner), PAP (project owner), AAP (contributor). Each tier gets a pass link. Paste it in Claude for Chrome and the right dashboard opens instantly.
Storage
Git-versioned file hosting
Every file upload is a commit. Roll back to any version. Promote from draft to live in one step. Optional GitHub mirror for public repos.
Hub features

What Hub does

Every project created on Hub gets the full stack automatically. No setup required.

Hosting
Project pages
Every project gets a public URL at hub.labs.co/{name}/. Static HTML, CSS, JS, media assets. Content is served from the live/ branch. Visitors see a welcome page by default, or your deployed app once Claude builds it.
Versioning
Git history & rollback
Every upload creates a commit. Every commit is addressable at /{name}/v/{N}/. Promote any version to live with one API call. Full history visible in the PAP dashboard. Optional GitHub auto-sync on every commit.
Dashboard
PAP — project dashboard
The project owner dashboard. Reachable by pass link or from @LabsHubBot. Shows bot status, subscriber count, broadcast UI, command editor, bot profile editor, webhook status, audience analytics, runtime logs, file versions, GitHub sync controls.
Collaboration
AAP — contributor access
The project owner can generate AAP links for collaborators. Each AAP gets an isolated branch aap/{id}/. Contributors upload and commit freely. The PAP owner merges and promotes to live. No credentials shared.
AI Build
Wizard & prompt seeding
The Hub Wizard (hub.labs.co/hub/wizard) opens inside Telegram as a Mini App. Answer 15 Yes/No/Skip questions about your project. Hub generates a structured build prompt and seeds it as prompt.md at your live URL. Open that URL in Claude for Chrome — Claude reads it and builds the project end-to-end.
Buffer
Personal cross-device buffer
Every message you send to @LabsHubBot (that isn’t a command) is auto-saved to your personal buffer. Server-side SQLite keyed by Telegram ID. Access from any device at hub.labs.co/hub/buffer?tg={id}. Search, copy link, clear. JSON feed available for use in projects.
Smart intake
Universal message router
Send anything to @LabsHubBot and it figures out what to do. Bot token → link & create project. API key → save to server config. URL → set as webhook. Project name → create project. Description sentence → generate plan via LLM. Wizard JSON → full project creation with prompt seeding.
Part 2 — Telegram

What Hub does with your Telegram bot

Send one token to @LabsHubBot. That is the only time you touch BotFather. Every feature below is available from the dashboard from that moment on.

Setup
Link & validate
Send your bot token to @LabsHubBot. Hub calls getMe to validate it, stores the token securely, creates your project, and returns a PAP dashboard link. The bot is immediately live in polling mode. No webhooks to configure, no servers to provision.
Messaging
Broadcast to all subscribers
Send a message to every subscriber at once from the dashboard. HTML formatting supported (<b>, <i>, <code>, links). After sending, shows delivered count and skipped count. Available in polling mode only.
Messaging
Send to specific user
Enter any Chat ID or @username and compose a message directly from the dashboard. No code. Useful for support replies, targeted notifications, testing. HTML formatting supported. Confirmation shows delivery status.
Commands
Edit bot command menu
Your bot’s command list is loaded live from Telegram when you open the dashboard. Edit inline: change command names, update descriptions, add new rows, delete existing ones. One click saves back to Telegram via setMyCommands. No BotFather needed.
Profile
Edit bot profile
Three fields, editable from the dashboard:
Display name — what users see as the bot’s name (up to 64 chars, setMyName)
Short description — shown in Telegram search results (up to 120 chars, setMyShortDescription)
About text — shown on the bot’s profile page (up to 512 chars, setMyDescription)
All three save in one click. Current bot name prefills the placeholder automatically.
Webhook
Webhook management
Full webhook visibility and control from the dashboard:
• Live status from getWebhookInfo — current URL, pending update count, last error message
• Set webhook — paste a URL, click “enable webhook”
• Delete webhook — one click, switches back to polling mode immediately
Polling ↔ webhook switch requires zero server restarts.
Subscribers
Subscriber tracking
/start in your bot adds the user as a subscriber. /stop removes them. The subscriber count shows live in the dashboard header. Used by broadcast to determine recipient list and skip count.
Analytics
Audience analytics
Automatically collected, no setup needed:
• Users — unique user count
• Events — total interaction count
• DAU 7d / DAU 30d — daily active users over last 7 and 30 days
• Languages — user language distribution with progress bars
• Countries — user country distribution with progress bars
• Peak hours — heatmap of activity by hour of day (UTC)
• Day of week — activity distribution by weekday
• Command usage — ranked table of which commands users call most
Logs
Runtime logs
View recent bot activity directly in the dashboard. Useful for debugging message handling and confirming deliveries. One-click clear. No SSH access needed.
Info
Live bot info
Refresh button in the profile section calls getMe live from Telegram and shows a toast with the current username, display name, and bot status. Confirms the token is still valid.
Unlink
Detach the bot
Remove the bot token from the project at any time. The project stays intact — files, versions, analytics history all remain. You can link a different token to the same project, or leave it as a pure hosting project.
Getting started

Five steps to a live project

From zero to a running bot with a dashboard, analytics, and a public page.

  1. Create a bot in @BotFather
    Open Telegram → @BotFather → /newbot. Pick a name and a username ending in bot. Copy the token it gives you.
  2. Send the token to @LabsHubBot
    Paste the token directly into the chat. Hub validates it with Telegram, creates your project instantly, and links the bot.
  3. Get your dashboard link (PAP)
    Hub replies with two buttons: Open (your public project page) and Dashboard (your private PAP link). Bookmark the dashboard link — it’s your control panel forever.
  4. Build with Claude (optional)
    Drop the project URL into Claude for Chrome. Claude reads the prompt.md that Hub seeded at your project URL and builds the full project — HTML, logic, bot commands — then uploads, commits, and promotes it to live automatically.
  5. Manage from the dashboard
    Broadcast, edit commands, update your bot’s name and description, switch to webhook mode, view analytics — all from the web, all without touching BotFather or writing code.
API reference

Bot management endpoints

All endpoints require PAP or SAP Bearer token. Callable by humans, agents, or any HTTP client.

GET/drafts/project/botBot status: attached, username, mode (polling/webhook)
PUT/drafts/project/botLink a bot token to this project

POST/drafts/project/bot/profileSet name, short description, about text
GET/drafts/project/bot/commandsGet current command list from Telegram

GET/drafts/project/bot/webhookLive getWebhookInfo — URL, pending count, last error
PUT/drafts/project/bot/webhookSet webhook URL
DEL/drafts/project/bot/webhookDelete webhook, switch to polling

POST/drafts/project/bot/sendSend message to specific chat_id or @username
GET/drafts/project/bot/logsRecent runtime log lines
DEL/drafts/project/bot/logsClear runtime logs
